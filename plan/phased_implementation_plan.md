# AI 论文与课件知识图谱学习平台分阶段实施方案

## 总体目标

把当前项目逐步建设成：

> 本地单用户 AI 论文与课件知识图谱管理系统 + NotebookLM 式学习材料生成平台

核心能力分三条线并行演进：

- 资料线：上传、解析、清洗、切块、索引
- 智能线：检索、问答、Notebook 生成、学习路径
- 知识线：实体、关系、图谱、证据追溯

## 阶段 0：原型闭环，已完成

目标：证明系统能跑通。

已完成：

- FastAPI 后端可启动
- 文档上传、解析、切块
- 本地轻量检索
- 基础图谱构建
- 问答接口
- Jupyter Notebook 生成
- 单元测试通过

验收状态：已通过。

## 阶段 1：稳定工程底座与文档解析质量

目标：让资料进入系统时是干净、可追踪、可复现的。

主要任务：

- 修复中文乱码与编码识别问题
- 增加 `/api/health` 健康检查接口
- 增加文档预览接口
- 增加解析状态与错误信息
- 把 `kg_store`、生成文件、缓存目录规范化
- 增加基础配置文件，例如数据目录、chunk 大小、Top-K 参数
- 补充 API 级测试

交付物：

- 稳定可启动的后端服务
- 干净的文档解析结果
- 可查看原文与 chunk 预览
- 明确的错误提示

验收标准：

- 上传 UTF-8 中文 Markdown 不乱码
- 上传 PDF/MD/TXT 后能解析并预览
- 解析失败时返回明确错误
- 单元测试和接口 smoke test 通过

## 阶段 2：NotebookLM 式学习材料生成升级

目标：让生成的 `.ipynb` 真正像学习材料，而不是简单拼接片段。

主要任务：

- 读取 `ref/*.ipynb` 作为风格样例
- 抽象 Notebook 模板：
  - 学习目标
  - 先修知识
  - 核心概念
  - 分节讲解
  - 代码示例
  - 练习题
  - 参考来源
- 增加“先生成大纲，再生成内容”的流程
- 接入 OpenAI-compatible LLM 配置，支持 DeepSeek/MiMo/OpenAI
- 增加生成参数：
  - 难度
  - 是否包含代码
  - 练习题数量
  - 输出语言
  - 学习目标类型

交付物：

- 更高质量的 `.ipynb`
- 可配置的 Notebook 生成接口
- 学习材料生成记录

验收标准：

- 生成 Notebook 能被 Jupyter 打开
- 结构接近 `ref` 中的样例
- 内容包含讲解、例子、练习、参考来源
- 对同一主题可重新生成不同版本

## 阶段 3：最小 Web 前端

目标：从“接口可用”变成“人能顺手使用”。

主要任务：

- 搭建前端页面
- 文档上传页
- 文档列表与解析状态
- 搜索与问答页面
- Notebook 生成页面
- Notebook 列表与下载
- 简单图谱节点浏览

交付物：

- 本地可访问 Web 应用
- 后端 API 全部接入前端

验收标准：

- 不用 Swagger 也能完成完整流程
- 上传文档、解析、搜索、生成 Notebook、下载 Notebook 全部可视化完成

## 阶段 4：语义检索升级

目标：提升检索和生成材料的准确性。

主要任务：

- 接入 embedding 模型，优先 `bge-m3`
- 本地向量索引从 TF-IDF 升级到 FAISS
- 后续预留 Qdrant
- 增加 hybrid search：
  - 关键词检索
  - 向量检索
  - 图谱实体召回
- 增加 reranker 预留接口

交付物：

- 语义检索能力
- 更准确的问答和 Notebook 资料召回

验收标准：

- 查询同义表达也能召回相关资料
- AI 术语、论文标题、课程概念能稳定检索
- Notebook 生成使用的资料更贴近主题

## 阶段 5：知识图谱增强

目标：让图谱从“实体索引”升级为“学习和研究的知识骨架”。

主要任务：

- 明确 V1 本体：
  - Paper
  - Courseware
  - Course
  - Concept
  - Method
  - Model
  - Task
  - Dataset
  - Metric
  - Chunk
- 增加实体消歧与别名表
- 增加关系抽取规则
- 增加图谱浏览接口
- 增加图谱导出 JSON/GraphML
- 后续可迁移 Neo4j

交付物：

- 更稳定的实体和关系
- 图谱查询与可视化数据接口

验收标准：

- `Transformer`、`Attention`、`BERT`、`GPT` 等实体能稳定识别
- Chunk 能关联到实体
- 问答结果能返回相关图谱路径

## 阶段 6：学习体验增强

目标：从“生成笔记”升级为“辅助学习”。

主要任务：

- 生成学习路线
- 生成复习题
- 生成 FAQ
- 生成术语表
- 生成论文阅读卡片
- 生成课程章节总结
- 支持“我学到哪了”的学习记录

交付物：

- 学习路线生成器
- 复习包生成器
- 论文卡片生成器

验收标准：

- 对一个主题能生成完整学习包
- 对一组论文能生成阅读顺序和关键问题
- 对一门课件能生成章节知识地图

## 阶段 7：产品化与长期演进

目标：让系统更接近长期可用工具。

主要任务：

- 用户配置管理
- 数据备份与恢复
- 批量导入
- 任务队列
- 日志与可观测性
- Docker 化
- 可选多用户权限
- Qdrant / Neo4j / MinIO 替换本地存储

交付物：

- 可部署版本
- 稳定数据目录
- 可迁移架构

验收标准：

- 换机器后可以恢复数据
- 长文档批量处理不卡主流程
- 存储层可替换

## 推荐实施顺序

建议接下来按以下顺序做：

1. 阶段 1：稳定工程底座与文档解析质量
2. 阶段 2：Notebook 生成质量升级
3. 阶段 3：最小 Web 前端
4. 阶段 4：语义检索升级
5. 阶段 5：知识图谱增强

第一步建议从阶段 1 开始，因为它会直接解决乱码、预览、错误处理和配置问题。这个地基稳了，后面的 Notebook 质量和前端体验都会更顺。
