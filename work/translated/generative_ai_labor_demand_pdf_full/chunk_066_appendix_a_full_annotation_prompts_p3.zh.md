# 附录A 完整注释提示

你将收到一个JSON对象，包含：
- posting_id: 字符串。职位发布ID。
- posting_title: 字符串。职位名称。
- tasks: 任务对象列表，每个任务包含：
  - task_id: 字符串（例如"t1"）
  - task: 描述工作者任务活动的字符串。

请为每个任务分配一个且仅一个暴露标签，标签取自 {"E0", "E1", "E2"}：

E0（无暴露）：
- 如果现有的GenAI/LLM工具（ChatGPT、辅助编程工具、ASR/TTS、多模态工具）无法以同等质量将任务时间减少50%，则标记为E0。
- 需要大量面对面互动、物理操作、检查、维修、现场护理或受资格认证约束决策的任务必须为E0。

E1（直接暴露）：
- 如果单个现成的GenAI/LLM工具（无需特殊集成或微调）能够以同等质量将任务工作量减少50%，则标记为E1。
- 典型的E1模式（非详尽）：
  * 编写或转换文本或代码
  * 翻译、语气/风格编辑
  * 对中等长度文档进行摘要；提取结构化信息
  * 根据提供的内容起草电子邮件、简报、幻灯片、问答
  * ASR转录；TTS草稿旁白；简单的音频清理
  * 简单的图像生成/编辑（缩略图、字幕、营销图片）
- 不适用E1的情况：需要内部数据访问、企业系统、自定义工作流或超出复制粘贴范围的自动化任务。

E2（LLM+暴露）：
- 如果单个现成的GenAI/LLM工具本身无法实现50%的时间减少，但一个浅层的AI驱动软件层有可能实现这一目标，则标记为E2。
- 浅层软件层的示例：
  * 对私有/内部文档的检索引擎或搜索
  * 多步骤工作流（草稿→审查→格式化→上传）
  * 产品内助手（CRM、IDE、帮助台）读取上下文并建议操作/回复
  * 客服系统中的自动建议回复
  * 评分标准/政策/品牌执行；轻量级微调
  * 集成到资产库或翻译流程的多模态工具
- 不适用E2的情况：
  * 如果已适用E1 → 选择E1
  * 即使通过集成，AI也无法合理地将时间减半 → 选择E0
  * 需要大量定制机器学习、机器人或新模型训练 → 选择E0

输出格式（严格）：
返回一个JSON对象：
{
  "posting_id": "<与输入相同>",
  "task_exposures": [
    {"task_id": "t1", "exposure_label": "E1"},
    {"task_id": "t2", "exposure_label": "E0"}
  ]
}

规则：
- 仅返回有效JSON。不包含解释或额外文本。
- 每个输入task_id必须恰好出现一次。
- exposure_label必须是"E0"、"E1"或"E2"中的一个。
