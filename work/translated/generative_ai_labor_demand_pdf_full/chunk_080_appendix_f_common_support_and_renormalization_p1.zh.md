# 附录F 共同支撑与重标准化

分解中的一个实际问题是，观测到的职业×资历×行业单元格集合会随时间变化。某些单元格在2021年基期和t期均能观测到，而另一些则仅出现在其中一期。这种支撑集不重叠会机械性地影响分解，尤其是在三重分解设定中，部分变化可能被交互项吸收。为解决此问题，我们定义2021年与t期之间的时期特定共同支撑集为 S = {c: w > 0 且 w > 0}，其中 t 表示时期，c 表示单元格。因此，S_t 仅包含基期和当前期均能观测到的单元格。将关注范围限制在 S_t 后，我们在共同支撑样本内重新标准化职位发布份额。具体地，对于每个 c ∈ S_t，定义：

w̃_ct^(t) = w_ct / Σ_{j∈S_t} w_jt,  w̃_c,2021^(t) = w_c,2021 / Σ_{j∈S_t} w_j,2021

这些重新标准化的权重在各期共同支撑内之和为1：
Σ_{c∈S_t} w̃_ct^(t) = 1, Σ_{c∈S_t} w̃_c,2021^(t) = 1.

使用这些权重，t期的共同支撑加总暴露度为：
Ē_CS,renorm_t = Σ_{c∈S_t} w̃_ct^(t) E_ct

对应的基期对象为：
Ē_CS,renorm_2021^(t) = Σ_{c∈S_t} w̃_c,2021^(t) E_c,2021

重标准化之所以必要，是因为若不进行此操作，基于共同支撑的分解仍会反映持久单元格之间的变化，以及共同支撑上总职位发布质量的变化。通过重标准化，我们分离出两个时期均能观测到的单元格之间的变化。一般而言，重标准化后的共同支撑对象与原始加总暴露度并不相同。因此，我们将原始加总与重标准化共同支撑加总之间的差异报告为支撑集不重叠的概括性度量：

差距_t = Ē_t - Ē_CS,renorm_t

对于基期，可使用 Ē_CS,renorm_2021^(t) 进行类似比较。

我们还报告两个简单的重叠诊断指标：
mcur_t = Σ_{c∈S_t} w_ct, mbase_t = Σ_{c∈S_t} w_c,2021

其中，mcur_t 是当前期职位发布质量落在共同支撑上的份额，mbase_t 是2021年基期的相应份额。这些度量提供了跨两个时期职位发布分布可比性的透明概括。

F.1 重标准化共同支撑分解的诊断结果

【本段术语】

- support non-overlap: 支撑集不重叠
- three-fold specification: 三重分解设定

【本段要点】
- 分解中单元格集合随时间变化导致支撑集不重叠，影响分解结果，尤其是三重分解中的交互项。
- 定义共同支撑集为两个时期均出现的单元格，并通过重标准化权重消除总发布质量变化的影响，仅保留单元格间变化。
- 重标准化后的共同支撑暴露度与原始加总不同，其差异衡量支撑集不重叠的程度。
- 提出两个重叠诊断指标（当前期和基期在共同支撑上的发布份额），总结跨时期可比性。
- 附录F.1将展示重标准化共同支撑分解的诊断结果。
